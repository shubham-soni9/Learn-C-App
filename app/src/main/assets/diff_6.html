Serial No. Basis of distinction Array Structure
1. Basic An array is a collection of variables of same data type. A structure is a collection of variables of different data type.
2. Syntax type array_name[size]; struct sruct_name{
type element1;
type element1;
.
.
} variable1, variable2, . .;
3. Access Array elements are accessed by their index number. Structure elements are accessed by their names.
4. Operator Array declaration and element accessing operator is "[ ]" (square bracket). Structure element accessing operator is "." (Dot operator).
5. Pointer Array name points to the first element in that array so, array name is a pointer. Structure name does not point to the first element in that structure so, structure name is not a pointer.
6. Objects Objects (instances) of an array can not be created. Structure objects (instance or structure variable) can be created.
7. Size Every element in array is of same size. Every element in a structure is of different data type.
8. Bit filed Bit filed can not be defined in an array. Bit field can be defined in a structure.
9. Keyword There is no keyword to declare an array. "struct" is a keyword used to declare the structure.
10. User-defined Arrays are not user-defined they are directly declared. Structure is a user-defined datatype.
11. Accessing Accessing array element requires less time. Accessing a structure elements require comparatively more time.
12. Searching Searching an array element takes less time. Searching a structure element takes comparatively more time than an array element.
13. Memory Array elements are stored in contiguous memory location. Structure elements may not be stored in a contiguous memory location.
